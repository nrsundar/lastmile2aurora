How to Use Performance Testing
Getting Started
  1. Setup Test Data: Click "Setup Test Data" to create sample tables with 1,000 customers, 500 products, and 5,000 orders
  2. Choose Iterations: Select how many times each query runs (more iterations = more accurate results)
  3. Run Tests: Click "Run Performance Tests" to compare original vs optimized SQL queries
  4. View Results: Analyze timing improvements and execution plans
  5. Export Results: Download detailed performance data for further analysis
What Gets Tested
  • JOIN Syntax: Implicit comma joins vs explicit INNER JOIN
  • Subquery Optimization: IN subqueries vs EXISTS clauses
  • Aggregation Performance: Different approaches to GROUP BY with JOINs
Example Test Cases
Original (Slower)
-- Implicit join
SELECT c.name, o.total 
FROM customers c, orders o
WHERE c.id = o.customer_id
Optimized (Faster)
-- Explicit join
SELECT c.name, o.total 
FROM customers c
INNER JOIN orders o ON c.id = o.customer_id
Performance Tips
  • Green badges show performance improvements
  • Red badges indicate slower performance
  • Use 3+ iterations for reliable results
  • Check execution plans for detailed insights
  • Results may vary based on data size and indexes
Important Notes
  • Set up test data before running tests
  • Performance results are environment-specific
  • Always test with production-like data volumes
  • Consider database server configuration
