EXPLAIN ANALYZE Documentation

Try EXPLAIN Analysis

Overview

EXPLAIN ANALYZE Analysis automatically parses execution plans to identify performance bottlenecks and provides specific optimization recommendations.

What It Analyzes:
  • Execution time and planning overhead
  • Sequential vs. index scan usage
  • Join strategy efficiency
  • Sort operation performance
  • Buffer hit ratios and I/O patterns
Benefits:
  • Identifies specific performance issues
  • Provides actionable optimization steps
  • Supports multiple database platforms
  • Generates index recommendations
  • Performance scoring and grading

How to Get EXPLAIN ANALYZE Output

PostgreSQL
-- Basic format EXPLAIN (ANALYZE, BUFFERS) SELECT * FROM table_name WHERE condition; -- JSON format (recommended) EXPLAIN (ANALYZE, BUFFERS, FORMAT JSON) SELECT * FROM table_name WHERE condition; -- Text format with detailed info EXPLAIN (ANALYZE, BUFFERS, VERBOSE, FORMAT TEXT) SELECT * FROM table_name WHERE condition;
Tip: Use BUFFERS for I/O statistics and JSON format for detailed parsing.
Oracle
-- Enable timing statistics ALTER SESSION SET TIMED_STATISTICS = TRUE; -- Explain the plan EXPLAIN PLAN FOR SELECT * FROM table_name WHERE condition; -- Display the execution plan SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY( NULL, NULL, 'ALLSTATS LAST')); -- Alternative with actual execution SELECT * FROM TABLE(DBMS_XPLAN.DISPLAY_CURSOR( NULL, NULL, 'ALLSTATS LAST'));
Note: Include ALLSTATS for comprehensive statistics.
SQL Server
-- Enable execution statistics SET STATISTICS IO ON; SET STATISTICS TIME ON; -- Execute your query SELECT * FROM table_name WHERE condition; -- Or use execution plan SET SHOWPLAN_ALL ON; SELECT * FROM table_name WHERE condition; SET SHOWPLAN_ALL OFF; -- For actual execution plan SET STATISTICS PROFILE ON; SELECT * FROM table_name WHERE condition; SET STATISTICS PROFILE OFF;
Tip: Combine IO and TIME statistics for complete analysis.

Step-by-Step Usage Guide

Step 1: Execute EXPLAIN ANALYZE on Your Query

Run the EXPLAIN ANALYZE command on your problematic query in your database:

  • Include actual execution (ANALYZE option)
  • Add buffer statistics if available
  • Use verbose output for detailed information
  • Copy the complete output including timing
Step 2: Prepare Your Query and Output

Gather both pieces of information:

  • Original Query: The exact SQL statement that was explained
  • EXPLAIN Output: Complete execution plan with timing information
  • Database Type: PostgreSQL, Oracle, or SQL Server
Step 3: Upload to Analysis Tool
  • Navigate to EXPLAIN Analysis page
  • Select your source database type
  • Paste the original query in the first text area
  • Paste the complete EXPLAIN output in the second area
Step 4: Review Analysis Results

The analysis will provide:

  1. Performance Score: Overall grade (A-F) with specific metrics
  2. Performance Issues: Identified bottlenecks with severity levels
  3. Optimization Suggestions: Actionable recommendations by category
  4. Index Recommendations: Specific indexes with DDL statements
Step 5: Implement Recommendations

Apply optimizations in order of priority:

  • Start with high-severity issues first
  • Test index recommendations in development
  • Monitor performance improvements
  • Re-run EXPLAIN ANALYZE to verify improvements
Performance Thresholds
High Severity:
  • Execution time > 1 second
  • Sequential scans on large tables
  • Expensive nested loops
Medium Severity:
  • Planning time > 100ms
  • Multiple sort operations
  • Suboptimal join strategies
Low Severity:
  • Minor inefficiencies
  • Optimization opportunities
  • Best practice recommendations

Troubleshooting & Best Practices

Common Issues & Solutions:
Error: "Invalid EXPLAIN format"

Solution: Ensure you've copied the complete EXPLAIN output including timing information and execution statistics.

Error: "Query text is required"

Solution: Provide the original SQL query that was used to generate the EXPLAIN output.

Error: "Parsing error"

Solution: Check for unsupported EXPLAIN format variations. Use standard EXPLAIN ANALYZE output.

No Recommendations Generated

Reason: Query may already be well-optimized or the execution plan doesn't show obvious bottlenecks.

Best Practices:
Before Analysis:
  • Run ANALYZE on all tables
  • Use representative data volumes
  • Execute queries multiple times for consistency
  • Clear cache for accurate I/O measurements
After Analysis:
  • Test recommendations in development first
  • Implement one change at a time
  • Monitor performance impact
  • Re-analyze after optimizations
Performance Monitoring:
  • Track query execution trends
  • Monitor index usage statistics
  • Regular EXPLAIN ANALYZE runs
  • Set up performance alerts
